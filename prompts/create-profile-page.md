# Задача: реализовать страницу профиля пользователя

## Контекст проекта

Стек: Vite + React 18 + TypeScript, SCSS (CSS-модули или plain SCSS), Zustand v5.

Дизайн-система: CSS-переменные объявлены в `client/src/App.scss` — **не вводи новые цвета**, используй только существующие переменные (например `--accent`, `--bg-input`, `--text-primary`, `--border-input`, `--text-secondary`, `--accent-dim` и т.д.). Тема переключается через `[data-theme="light"]`, переменные уже покрывают оба режима.

Роутинг настроен так, что `client/src/pages/ProfilePage/Profile.tsx` является заглушкой — именно его нужно реализовать.

Zustand-стор: `client/src/store/chatStore.ts`, хук `useChatStore`.
Доступные из стора данные:
- `user: User | null` — `{ id, email, balance, balanceUSD, createdAt }` (поля `balance`/`balanceUSD` — строки/числа, баланс в рублях и долларах соответственно)
- `chats: Chat[]` — массив чатов; каждый чат содержит `messages: Message[]`
- `models: Model[]` — загруженные модели

Тип `Message` (`client/src/types/index.ts`):
```ts
interface Message {
  id: string;
  role: 'user' | 'assistant';
  name: string;
  content: string;
  model?: string;   // id модели (только у assistant-сообщений)
  inContext: boolean;
  createdAt: Date;
}
```

## Что нужно реализовать

### 1. Структура страницы

Контейнер с `max-width: 800px`, центрированный по горизонтали, с комфортными отступами (padding ~24px). Страница скроллится вертикально при необходимости.

### 2. Приветствие

Вверху страницы — заголовок вида:
```
Привет, Artem
```
Имя берётся из localStorage по ключу `vakachat_display_name`. Если не задано — показывать часть email до `@` (из `user.email`). Стиль заголовка: крупный шрифт, акцентный цвет (`--accent`) на имени.

### 3. Блок «Настройки»

Секция с заголовком «Настройки». Содержит два поля:

**Как к вам обращаться**
- Label: «Отображаемое имя»
- Input type=text
- Placeholder: «Например, Артём»
- Значение читается из / сохраняется в `localStorage` по ключу `vakachat_display_name`
- Сохранение — кнопка «Сохранить» или автосохранение по `onBlur`

**Системный промпт**
- Label: «Системный промпт»
- Hint под label: «Отправляется нейросети перед каждым новым чатом»
- Textarea с авторесайзом (min-height ~80px)
- Placeholder: «Например: Отвечай кратко и по делу. Используй markdown.»
- Значение читается из / сохраняется в `localStorage` по ключу `vakachat_system_prompt`
- Сохранение — кнопка «Сохранить» рядом или автосохранение по `onBlur`

> Примечание: в этой задаче backend не затрагивается. Настройки хранятся только в `localStorage`. Системный промпт будет использован другими компонентами в будущем — просто сохраняй/читай его.

### 4. Блок «Статистика»

Секция с заголовком «Статистика». Данные считаются из `chats` стора (только то, что уже загружено на клиенте).

Отображай карточки/ряды со следующими метриками:

| Метрика | Как считать |
|---|---|
| **Чатов создано** | `chats.length` |
| **Сообщений отправлено** | сумма `role === 'user'` сообщений по всем чатам |
| **Ответов получено** | сумма `role === 'assistant'` сообщений по всем чатам |
| **Моделей использовано** | количество уникальных `message.model` среди assistant-сообщений |
| **Аккаунт создан** | `user.createdAt`, отформатированный как `дд.мм.гггг` |
| **Баланс** | `user.balanceUSD` в долларах и `user.balance` в рублях |

Карточки располагаются в сетке: 2–3 колонки на десктопе, 1–2 на мобильном. Каждая карточка: фон `--bg-input`, скруглённые углы, отступы, значение крупным шрифтом, подпись мелким `--text-secondary`.

> Если `chats` пустые или данных нет — показывай `0` / «—», не падай с ошибкой.

### 5. Стили

Создай файл `client/src/pages/ProfilePage/Profile.module.scss` (CSS-модули, как в AuthForm).

**Обязательные требования к стилям:**
- Только существующие CSS-переменные из `App.scss`, никаких хардкод-цветов
- Мобильная адаптация: брейкпоинт `@media (max-width: 640px)` — уменьшить отступы, сетка в 1 колонку, шрифты чуть меньше
- Инпуты и textarea должны выглядеть консистентно с остальным приложением: фон `--bg-input`, граница `--border-input`, цвет текста `--text-primary`, `border-radius: 8px`, `padding: 10px 14px`
- Focus-состояние: `border-color: var(--accent)`, `box-shadow: 0 0 0 3px var(--accent-focus-ring)`

### 6. Типизация и качество кода

- Весь новый код на TypeScript, без `any`
- Не добавляй лишних зависимостей — только то, что уже установлено в проекте
- Не изменяй store, types или другие существующие файлы — только `Profile.tsx` и новый `Profile.module.scss`
